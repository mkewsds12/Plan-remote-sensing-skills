# 空天地一体化监测方案自动生成工具

一句话输入行业领域（可选地区），自动生成完整的空天地一体化监测技术方案（Markdown + Word）。

## 快速开始

### 1. 安装依赖

```bash
pip install python-docx
```

### 2. 使用方式

在 VS Code 的 Copilot Chat 中输入：

```
/write-proposal 林业生态监测
```

也可以指定地区，生成贴合当地实际的方案：

```
/write-proposal 贵州省林业生态监测
```

然后等待即可。Copilot 会自动完成以下全部步骤：

1. 根据你的领域生成完整技术方案（约 2~3 万字）
2. 分章节写入 `.md` 文件（避免单次输出过大导致超时）
3. 调用脚本自动转换为排版好的 `.docx` 文件
4. 通知你文件路径

**你不需要手动复制、保存或运行任何命令。**

### 3. 输出文件

生成完成后，在工作区根目录下会看到两个文件：

```
# 不指定地区
林业生态监测解决方案.md      ← Markdown 源文件
林业生态监测解决方案.docx    ← 排版好的 Word 文档

# 指定地区
贵州省林业生态监测解决方案.md
贵州省林业生态监测解决方案.docx
```

### 地区支持说明

- **不带地区**（如 `林业生态监测`）：生成通用方案，不涉及具体地理信息
- **带地区**（如 `贵州省林业生态监测`）：方案将融入该地区的地理气候、行政区划、产业特征等信息，并在第一章增加「区域概况」小节

## 写作风格

生成的方案遵循严格的专业写作规范：

- **严谨科学**：所有数据标注来源，技术描述准确，不使用模糊或夸大用语
- **反 AI 痕迹**：不使用"此外""值得注意的是""总之"等 AI 常见过渡词，不使用排比句和感叹号
- **公文风格**：段落之间自然衔接，句式长短交替，符合技术方案的行文习惯
- **数据驱动**：关键参数给出具体数值范围，引用真实政策文件和技术标准

### 格式规范

生成时自动执行以下格式约束：

- **加粗标题后用冒号**：`**标题**：正文内容`，禁止用句号分隔
- **段落和列表项必须以句号结尾**：每段话、每个列表项的末尾都有句号
- **政策背景用段落格式**：按国家→省→市层级递进，每级一段完整的话，不使用项目符号列表

### 应用场景

- 应用场景不少于 5 个，每个场景不少于 500 字
- 每个场景包含：业务背景 → 工作流程 → 应用价值
- 应用价值是重点，必须详实说明管理效益、效率提升和决策支撑价值

## 方案结构

生成的方案包含 8 个章节：

```
1. 总体规划与建设目标
   ├── 项目背景（政策+趋势+痛点）
   ├── 项目必要性
   ├── 需求分析（业务/功能/性能）
   └── 建设目标

2. 监测体系技术框架
   ├── 天基卫星遥感监测
   ├── 空基无人机监测
   ├── 地基地面监测体系
   └── 天空地协同机制

3. 平台建设技术框架
   ├── 感知层
   ├── 数据层
   ├── 平台层
   └── 应用层

4. 应用场景（≥5个行业典型场景，每个≥500字）

5. 关键技术与创新点

6. 项目实施方案

7. 成果总览

8. 效益分析（经济/社会/生态）
```

## 方案规模

默认生成中型规模方案（全部 8 个章节，约 1.5~3 万字）。可以在输入时指定规模：

```
/write-proposal 林业生态监测，小型方案
/write-proposal 矿产矿区监测，中型方案（默认）
/write-proposal 水利水资源监测，大型方案
```

| 项目规模 | 包含章节 | 预估字数 |
|---------|---------|---------|
| 小型（百万级以下） | 1+2+3+4+8，省略5/6/7 | 8000-15000字 |
| 中型（百万-千万级，默认） | 全部章节 | 15000-30000字 |
| 大型（千万级以上） | 全部章节+附件（技术参数表、设备清单） | 30000-50000字 |

## Word 排版规格

自动生成的 Word 文档遵循公文排版标准：

| 项目 | 样式 |
|------|------|
| 正文字体 | 仿宋 四号（14pt） |
| 正文英文 | Times New Roman（仅英文字母和数字） |
| 中文标点 | 跟随中文字体（仿宋），双引号""不使用 Times New Roman |
| 标题字体 | 黑体 加粗 |
| 正文加粗 | 仿宋 加粗（非黑体） |
| 行距 | 28磅固定值 |
| 首行缩进 | 2字符 |
| 一级标题 | 居中，小二号（18pt） |
| 二级标题 | 左对齐，三号（16pt） |
| 三级标题 | 左对齐，小三号（15pt） |
| 四级标题 | 左对齐，四号（14pt） |
| 页边距 | 上3.7 / 下3.5 / 左2.8 / 右2.6 cm |
| 表格字体 | 仿宋 小四号（12pt） |
| 表格对齐 | 水平居中 + 垂直居中 |

## 单独使用转换脚本


## 自定义修改

### 调整 Word 排版样式

编辑 `.github/skills/write-proposal/scripts/md2docx.py` 顶部的配置：

```python
FONT_CONFIG = {
    "title_cn": "方正小标宋简体",  # 封面标题字体
    "heading_cn": "黑体",          # 标题字体
    "body_cn": "仿宋",             # 正文字体
    "body_en": "Times New Roman",  # 英文字体
    "font_size_body": Pt(14),      # 正文字号
    "line_spacing": Pt(28),        # 行距
    ...
}
```

### 调整方案提纲

编辑 `.github/skills/write-proposal/references/outline-guide.md`，修改标准提纲结构和各领域场景示例。

### 调整方案模板

编辑 `.github/skills/write-proposal/assets/proposal-template.md`，修改各章节的内容模板。

### 修改 Skill 行为

编辑 `.github/skills/write-proposal/SKILL.md`，可以调整：
- 写作规范和字数要求
- 各章节的写作要点
- 自动化流程步骤

## 常见问题

**Q: 生成过程中网络超时怎么办？**

已内置分章节写入机制，每次只写一个一级章节，正常不会超时。如果仍然发生，重新输入 `/write-proposal 领域名称` 即可。

**Q: Word 文件被占用无法覆盖？**

关闭正在打开的 Word 文件后重新运行，或者脚本会自动输出到带 `_v2` 后缀的新文件。

**Q: 想只生成部分章节？**

可以在输入时说明，例如：

```
/write-proposal 林业生态监测，只需要第1章和第4章应用场景
```

**Q: 生成的内容需要修改？**

直接编辑生成的 `.md` 文件，然后手动运行转换脚本重新生成 Word：

```bash
python .github/skills/write-proposal/scripts/md2docx.py 修改后的方案.md
```
